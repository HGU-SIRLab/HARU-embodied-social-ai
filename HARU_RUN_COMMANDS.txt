[ 2. HITL 데이터 수집 — 상세 사용법 ]
================================================================

목적: HARU가 잘못된 행동을 할 때 "이렇게 했어야 해"를 가르치는
      데이터를 쌓는 과정. 이 데이터로 LoRA 어댑터를 학습시킴.

------------------------------------------------------------
2-1. 실행
------------------------------------------------------------

  haru-ws && haru-hitl

  (내부적으로 4개 노드가 동시에 뜸)
    vision_node   — 카메라 프레임 발행
    action_node   — hitl_mode=true (hitl_node 명령 대기)
    brain_node    — Gemma 4 추론 (60초 주기)
    hitl_node     — 터미널 인터페이스 (포그라운드)

------------------------------------------------------------
2-2. 에피소드 흐름
------------------------------------------------------------

  [N] → 에피소드 시작 (새 episode_YYYYMMDD_HHMMSS/ 폴더 생성)
      ↓
  Gemma 4가 60초마다 장면을 보고 JSON 제안 출력
      ↓
  터미널에 제안 표시:
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   [에피소드 1 | 스텝 0] VLA 제안
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    표정  : empathy (5)
    감정  : empathy
    머리  : tilt=2400  pan=2057  roll=2041
    오른팔: 1738
    바퀴  : right=0  left=0
    발화  : 많이 힘드셨나요?
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   [A] 승인   [C] 교정   [S] 스킵   [E] 에피소드 종료
      ↓
  사람이 [A] / [C] / [S] 입력
      ↓
  [E] → 에피소드 저장 (metadata.json + step_XXXX.npz 파일들)

------------------------------------------------------------
2-3. 키 설명
------------------------------------------------------------

  [A] 승인
      - Gemma 4 제안이 적절할 때 사용
      - 제안된 JSON 그대로 로봇에 전송 & step_XXXX.npz 저장
      - is_corrected = False
      - 학습 효과: "이 상황에서 이 판단은 옳다" 강화

  [C] 교정  ← 핵심 기능
      - Gemma 4 판단이 틀렸을 때 사용
      - 교정 방법을 선택:

      [D] 직접 시연 (키네스테틱 티칭) ← 권장
          1. 토크 OFF → 관절이 자유롭게 움직여짐
          2. 손으로 로봇을 원하는 자세로 직접 조작
          3. Enter → 인코더 값 자동 캡처 후 토크 ON
          4. 표정 ID·발화·바퀴는 별도로 수동 입력
          ※ 관절 숫자를 몰라도 됨

      [M] 수동 입력 (숫자 직접 입력)
          아래 항목들을 직접 숫자로 입력 (Enter = 현재값 유지):
          · 표정 ID (0~7)
            0=neutral  1=joy  2=sadness  3=curiosity
            4=surprise  5=empathy  6=thinking  7=concern
          · 발화 텍스트 (한국어 문장 직접 입력)
          · head_tilt  (1500~3086, 중립 2048)  — 고개 끄덕임
          · head_pan   (1043~3071, 중립 2057)  — 고개 좌우
          · head_roll  (1630~2452, 중립 2041)  — 고개 기울기
          · r_arm_pitch     (1024~2451, 중립 1738)
          · l_arm_pitch     (37~1542,   중립 790)
          · r_shoulder_roll (1000~2050, 중립 1525)
          · r_elbow_pitch   (2047~3062, 중립 2555)
          · l_shoulder_roll (1047~2056, 중립 1552)
          · l_elbow_pitch   (1021~2007, 중립 1514)
          · right_wheel / left_wheel (-300~300, 이동 불필요시 0)

      - 교정된 값으로 로봇 실행 & 저장 (is_corrected = True)
      - 학습 효과: "이 상황에서 네 판단 대신 이렇게 해야 해" 교정

  [S] 스킵
      - 이 스텝은 데이터로 쓰기 애매할 때 사용
      - 로봇은 Gemma 4 제안대로 실행되지만 저장은 안 됨
      - 학습 데이터에 포함되지 않음

  [E] 에피소드 종료 & 저장
      - metadata.json 생성 후 에피소드 확정
      - 이후 새 에피소드 시작 가능 [N]

  [Q] 에피소드 취소
      - 현재 에피소드 전체 삭제 (data/episodes/에서 폴더 제거)
      - 잘못 수집됐다 싶을 때 사용

------------------------------------------------------------
2-4. 수집 전략
------------------------------------------------------------

  추천 에피소드 구성:
    - 에피소드 1개 = 5~20 스텝 (한 상호작용 맥락)
    - 첫 LoRA 학습 최소 요건: 20~30 스텝 이상 (교정 포함)
    - 교정 비율 높을수록 학습 신호 강함

  교정을 많이 하면 좋은 상황:
    - 상황에 어울리지 않는 표정 (슬픈 상황에 joy=1 출력 등)
    - 발화가 너무 짧거나 맥락과 무관할 때
    - 머리 방향이 상황과 안 맞을 때

  데이터 확인:
    ls -lh data/episodes/                    # 에피소드 목록
    ls data/episodes/episode_XXXXXX/         # 스텝 파일 목록
    python -c "
    import numpy as np
    d = np.load('data/episodes/episode_XXXXXX/step_0000.npz', allow_pickle=False)
    print({k: d[k].shape if hasattr(d[k], 'shape') else d[k].item() for k in d})"

[ 3. LoRA 학습 (에피소드 수집 후) ]
================================================================

------------------------------------------------------------
3-1. 학습 흐름 개요
------------------------------------------------------------

  에피소드 수집 (HITL)
        ↓
  data/episodes/  ← step_XXXX.npz 누적
        ↓
  python scripts/train_lora.py
        ↓
  data/adapters/adapter_YYYYMMDD_HHMMSS/  ← LoRA 어댑터 저장
        ↓
  다음 haru-brain 실행 시 자동 로드
  "[Brain] LoRA 어댑터 로드 완료: adapter_XXXXXXXX" 로그 확인

------------------------------------------------------------
3-2. 학습 실행 명령
------------------------------------------------------------

  source /home/herobot/robot_brain_workspace/haru_vla_env/bin/activate
  cd /home/herobot/robot_brain_workspace

  # 데이터 파싱 먼저 확인 (모델 로드 없음, 빠름)
  python scripts/train_lora.py --dry-run

  # 기본 학습 (rank 16, 3 epoch, lr 2e-4)
  python scripts/train_lora.py

  # 강한 학습 (데이터 많을 때)
  python scripts/train_lora.py --epochs 5 --rank 32

  # 전체 옵션
  python scripts/train_lora.py \
    --epochs   3      \   # 반복 횟수 (기본 3)
    --rank     16     \   # LoRA rank (기본 16, 클수록 표현력↑ 메모리↑)
    --alpha    32     \   # LoRA alpha = rank × 2 권장
    --lr       2e-4   \   # 학습률 (진동 시 1e-4로 낮추기)
    --grad-acc 4          # gradient accumulation step (기본 4)

------------------------------------------------------------
3-3. LoRA가 하는 일
------------------------------------------------------------

  Gemma 4 12B 베이스 모델 (22GB, 고정)
        +
  LoRA 어댑터 (~수십 MB, 학습됨)
        =
  "이 사용자·이 환경에서의 행동 편차"

  - 베이스 모델 가중치는 절대 변경 안 됨 → 파국적 망각 방지
  - 어댑터만 교체하면 여러 사용자/환경에 독립적으로 적용 가능
  - 새 데이터 수집 → 새 어댑터 학습 → 자동 교체 (이전 어댑터 보존)

------------------------------------------------------------
3-4. 어댑터 관리
------------------------------------------------------------

  ls data/adapters/                         # 생성된 어댑터 목록
  cat data/adapters/adapter_XXXX/README.md  # 해당 어댑터 학습 정보
  # brain_node는 항상 가장 최근 어댑터를 자동 선택

  # 특정 어댑터 이전 버전으로 롤백하려면:
  # adapter_manager.py의 latest_adapter()가 mtime(수정 시간) 기준으로 최신 선택하므로
  # 원하지 않는 최신 어댑터 폴더의 타임스탬프를 과거로 바꾸거나 이름 앞에 _ 붙이면 됨
  mv data/adapters/adapter_NEW data/adapters/_disabled_adapter_NEW

[ 6. 토픽 구조 ]
================================================================

  [카메라] → haru_vision → /haru_vision/compressed
  [마이크] → haru_audio  → /haru_audio/raw (16kHz float32)
                         → /haru_audio/vad (Bool)

  haru_brain (System 2, Gemma 4 12B)
    입력: /haru_vision/compressed, /haru_audio/raw
    출력: /haru_vla_raw        (JSON, 항상 여기만 발행)
          /haru_expression     (Int32)
          /haru_speech         (String)

  라우팅:
    직결 모드 (기본):  /haru_vla_raw → action_node
    HITL 모드:        /haru_vla_raw → hitl_node → /haru_system1_command → action_node

  action_node 파라미터:
    hitl_mode:=false  직결 (기본)
    hitl_mode:=true   HITL (haru-hitl 실행 시 자동 설정)

[ 2026-06-19 버그수정 이력 ]
================================================================

  hitl_node   : _kb_worker ↔ _run_correction stdin 경쟁 수정
                (select() 기반 타임아웃 폴링 + _in_correction 플래그)

  gemma4_inf  : JSON 파서 개선 (thought 채널 대응, 역순 탐색)
  train_lora  : total_steps=max(1,…) 보호, bytes/str 안전 디코딩
