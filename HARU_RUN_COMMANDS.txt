[ 1. 실행 모드 선택 ]
================================================================

■ 일반 운용 모드 (Triple-System 전체)
  haru-ws && haru-all
  (내부: vision + attention + brain + audio + action)

■ HITL 데이터 수집 모드
  haru-ws && haru-hitl
  (내부: vision + attention + brain + audio + action(hitl) + hitl_logger)

■ 개별 노드 단독 실행 (디버깅용)
  haru-vision      # 카메라만
  haru-action      # 모터만
  haru-brain       # Gemma 4만 (모델 로드 ~24초)
  haru-audio       # 마이크 VAD만

■ 주의 노드 단독 테스트 (Phase 5.5 신규)
  ./launch_vla.sh attention_only
  # /haru_attention/event 토픽으로 상황 분류 확인

[ 2. Triple-System 동작 원리 ]
================================================================

System 3 (haru_attention) — 항상 켜진 경량 지각층:
  · 카메라 프레임 → OpenCV 얼굴 감지 (~5ms, CPU)
  · 프레임 차분 → 모션 감지
  · /haru_audio/vad 구독 → 발화 여부 인지
  · 5-State FSM으로 상황 분류:

    상태          의미                    brain 트리거
    EMPTY         방에 아무도 없음         없음
    APPEARED      사람 등장 (5초 대기)     없음 (먼저 말 걸 기회)
    APPEARED→     5초 지나도 침묵         즉시 트리거
    PRESENT_SILENT
    CONVERSING    대화 중                  VAD 완료 시 즉시
    PRESENT_SILENT 있는데 안 말함          120s 주기

System 2 (haru_brain) — 이벤트 드리븐 판단층:
  · /haru_attention/event 구독 (60초 타이머 제거됨)
  · 상황 컨텍스트를 Gemma 4 프롬프트에 포함
  · Gemma 4가 말할지 침묵할지 자율 결정

  예시 컨텍스트:
    "사람이 7초 전에 등장했고 아직 말을 걸지 않음. 먼저 인사할지 판단하세요."
    "사용자가 방금 말을 마쳤음. 자연스럽게 응답하세요."
    "10분째 상호작용 없음. 조심스럽게 안부를 물을지 판단하세요."

Gemma 4 침묵 선택 (Phase 5.5 신규):
  · speech: "" 출력 시 → 발화 없음, 몸짓만 실행
  · 예: 사람 막 등장 → 고개 돌려 시선 맞추되 발화 없음 (자연스러운 비언어 반응)


================================================================
[ 3. HITL 데이터 수집 — 상세 사용법 ]
================================================================

목적: HARU가 잘못된 행동을 할 때 "이렇게 했어야 해"를 가르치는
      데이터를 쌓는 과정. 이 데이터로 LoRA 어댑터를 학습시킴.
      Phase 5.5부터: 상황 상태도 함께 저장 → 풍부한 학습 신호

------------------------------------------------------------
3-1. 실행
------------------------------------------------------------

  haru-ws && haru-hitl

  (내부적으로 6개 노드가 동시에 뜸)
    vision_node    — 카메라 프레임 발행
    attention_node — 상황 분류 + brain 트리거 (Phase 5.5 신규)
    audio_node     — 마이크 VAD
    action_node    — hitl_mode=true
    brain_node     — Gemma 4 추론 (attention event 기반)
    hitl_node      — 터미널 인터페이스 (포그라운드)

------------------------------------------------------------
3-2. 에피소드 흐름
------------------------------------------------------------

  [N] → 에피소드 시작
      ↓
  attention_node가 상황을 감지하면 brain_node 트리거
  (사람 등장 5초 후, 또는 VAD 완료 시)
      ↓
  Gemma 4가 상황 컨텍스트 포함해 JSON 제안 출력
      ↓
  터미널에 제안 표시:
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   [에피소드 1 | 스텝 0] VLA 제안 [상황: APPEARED_7s]
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    표정  : empathy (5)
    감정  : empathy
    머리  : tilt=2400  pan=2057  roll=2041
    오른팔: 1738
    바퀴  : right=0  left=0
    발화  : 많이 힘드셨나요?   (speech="" 이면 "(침묵 — 몸짓만)")
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   [A] 승인   [C] 교정   [S] 스킵   [E] 에피소드 종료
      ↓
  [E] → 에피소드 저장 (metadata.json + step_XXXX.npz)

------------------------------------------------------------
3-3. 키 설명 (변경 없음)
------------------------------------------------------------

  [A] 승인  — Gemma 4 제안 그대로 실행 & 저장 (is_corrected=False)
  [C] 교정  — [D] 키네스테틱 티칭 또는 [M] 수동 입력
  [S] 스킵  — 로그 저장 안 함
  [E] 에피소드 종료 & 저장
  [Q] 에피소드 취소

------------------------------------------------------------
3-4. 수집 전략
------------------------------------------------------------

  추천 에피소드 구성:
    - 에피소드 1개 = 5~20 스텝
    - 첫 LoRA 학습 최소 요건: 20~30 스텝 이상 (교정 포함)
    - 교정 비율 높을수록 학습 신호 강함

  Phase 5.5 추가 전략:
    - 다양한 상황 상태에서 수집
      · 사람 처음 등장 시 (APPEARED→PRESENT_SILENT) 인사 교정
      · 대화 중 (CONVERSING) 응답 교정
      · 오래 있다가 (PRESENT_SILENT) 안부 교정
    - 상황별로 다른 행동이 저장됨 → LoRA가 상황-행동 패턴 학습

[ 6. 토픽 구조 (Phase 5.5 Triple-System) ]
================================================================

  [카메라] → haru_vision → /haru_vision/compressed
  [마이크] → haru_audio  → /haru_audio/raw (16kHz float32)
                         → /haru_audio/vad (Bool)

  haru_attention (System 3, 경량, 항상 실행)
    입력: /haru_vision/compressed, /haru_audio/vad
    출력: /haru_attention/event (String JSON)

  haru_brain (System 2, Gemma 4 12B)
    입력: /haru_attention/event (트리거 + 컨텍스트)
          /haru_audio/raw (오디오 데이터, VAD 트리거 시 첨부)
    출력: /haru_vla_raw (JSON)
          /haru_expression (Int32)
          /haru_speech (String)

  라우팅:
    직결 모드 (기본):  /haru_vla_raw → action_node
    HITL 모드:        /haru_vla_raw → hitl_node → /haru_system1_command → action_node

[ 8. attention_node 파라미터 조정 ]

  # 얼굴 등장 후 대기 시간 조정 (기본 5초)

  ros2 run haru_attention attention_node --ros-args -p motion_threshold:=5.0

  ros2 run haru_attention attention_node --ros-args -p face_lost_timeout:=3.0

  # 대화 종료 판정 침묵 시간 (기본 30초)
  ros2 run haru_attention attention_node --ros-args -p conversation_timeout:=20.0

[ 13. 추론 가속 (Phase 5.6) ]
================================================================

  brain_node.py는 시작 시 자동으로 아래 순서로 추론 경로 선택:
    1순위: TRT-LLM 서버 (포트 8000) → ~6-10s
    2순위: auto_round W4A16 양자화 모델 → ~10-20s
    3순위: HF bf16 GPU (기본값, 즉시 사용 가능) → ~15-30s

------------------------------------------------------------
13-1. auto_round W4A16 양자화 (1-3시간, 1회 실행)
------------------------------------------------------------

  source /home/herobot/robot_brain_workspace/haru_vla_env/bin/activate
  nohup python3 /home/herobot/robot_brain_workspace/scripts/quantize_gemma4_autoround.py \
    2>&1 | tee /tmp/quantize.log &
  echo "양자화 PID: $!"

  # 진행 상황 확인:
  tail -f /tmp/quantize.log

  # 완료 확인:
  ls /home/herobot/robot_brain_workspace/data/gemma4_autoround_w4a16/
  # → config.json, *.safetensors 파일이 있으면 완료

  완료 후: 다음 haru-brain 실행 시 자동으로 auto_round 경로 선택

------------------------------------------------------------
13-2. TRT-LLM Docker 빌드 (8-16시간, 1회 실행)
------------------------------------------------------------

  # 빌드 시작 (백그라운드)
  cd /home/herobot/herobot_ws/jetson-containers
  nohup bash /home/herobot/robot_brain_workspace/scripts/build_trtllm_docker.sh \
    2>&1 | tee /tmp/trtllm_build.log &

  tail -f /tmp/trtllm_build.log
  docker images | grep tensorrt   # 완료 시 이미지 출력

  # 빌드 완료 후 → 엔진 변환 (1-3시간):
  bash /home/herobot/robot_brain_workspace/scripts/convert_gemma4_trtllm.sh

  # 변환 완료 후 → 서버 실행:
  bash /home/herobot/robot_brain_workspace/scripts/run_trtllm_server.sh

  # 서버 확인:
  curl http://localhost:8000/v1/models
