[A-1] bitsandbytes 4-bit 비호환 (SM87)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
증상: Gemma 4 12B를 load_in_4bit=True로 로드하면 CUDA 커널 오류 발생
원인: bitsandbytes의 4-bit NF4 커널이 SM80(A100) 이상을 타겟으로 컴파일됨.
      Jetson AGX Orin은 SM87 → 커널 바이너리 없음
오류: "CUDA error: no kernel image is available for execution"
해결: 4-bit bitsandbytes 포기. 대안:
      - auto_round W4A16 (현재 사용 중 ✅)
      - TRT-LLM W4A16 (빌드 중 🔄)
영구 원칙: Gemma 4는 bf16(22GB) 또는 W4A16만 가능. bitsandbytes 4-bit 금지.

[A-2] gptqmodel torch 버전 비호환
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
증상: gptqmodel 설치 시 의존성 오류
원인: gptqmodel >= 1.x가 torch >= 2.8.0을 요구
      Jetson에서 사용 가능한 최신 NVIDIA wheel은 torch 2.5.0a0+nv24.08
오류: "Requires torch>=2.8.0, but torch 2.5.0 is installed"
해결: gptqmodel 사용 불가. auto_round 0.13.1로 대체.
주의: Jetson torch는 PyPI 표준 wheel 사용 불가 (CUDA 12.6 전용 NVIDIA wheel 필수)

[A-3] auto_round — Gemma 4 RoPE 차원 불일치
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
증상: auto_round 양자화 중 RuntimeError: shape mismatch
원인: Gemma 4 Unified는 이종(heterogeneous) 어텐션 구조 사용:
      - sliding_attention 레이어 (5개마다): head_dim=256, 로컬 RoPE
      - full_attention 레이어 (6번째마다): global_head_dim=512, 글로벌 RoPE
      auto_round 0.13.1이 블록 간 position_embeddings를 재사용 → 차원 불일치 (512 vs 256)
오류: "RuntimeError: shape '[batch, seq, 256, 2]' is invalid for input of size ..."
해결: apply_rotary_pos_emb monkey-patch 적용
      → scripts/quantize_gemma4_autoround.py에 패치 포함됨 ✅
파일: /home/herobot/robot_brain_workspace/scripts/quantize_gemma4_autoround.py

[A-4] auto_round — Jetson PyTorch 2.5 set_submodule 버그 (가장 중요!)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
증상: 양자화 "완료"처럼 보이지만 저장된 모델에 qweight가 없음 (손상된 모델)
      로드 후 실제 추론하면 원본 정밀도(fp32)로 동작 또는 AttributeError
원인: Jetson PyTorch 2.5의 nn.Module.set_submodule()이
      "type(mod) is not torch.nn.Module"로 exact type check를 수행.
      nn.Module의 모든 서브클래스(예: Linear, Conv2d)에서 AttributeError 발생.
      auto_round의 set_module()이 이 오류를 조용히 무시(silent no-op)하여
      QuantLinear가 모델에 설치되지 않음.
      이후 release_layer_safely()가 원본 nn.Linear의 weight를 None으로 파괴
      → 저장 파일에 qweight 없는 손상 모델 생성.
진단법: python3 -c "
    from auto_round.auto_quantizer import AutoRoundQuantizer
    # QuantLinear 설치 후 모델 내 qweight 키 개수 확인
    # 정상: ~328개, 비정상: 0개
"
해결: _patched_set_module() 패치:
      getattr/setattr 직접 탐색으로 set_submodule 우회
      scripts/quantize_gemma4_autoround.py 상단에 적용됨 ✅
검증: 328 qweight, 1333 키, 2 shard 정상 저장 확인됨.

[A-5] Gemma 4 — AutoModelForImageTextToText 클래스 오류
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[A-6] Gemma 4 — gradient_checkpointing use_reentrant 오류
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
